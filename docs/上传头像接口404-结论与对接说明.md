## 上传头像 404 问题结论与对接说明

### 结论
- 前端实际请求的是 `POST /api/upload`，而后端提供的上传图片接口是 `POST /api/upload/image`；路径不匹配导致 404。

### 后端现状
- 全局前缀：`/api`
- 上传控制器：`@Controller('upload')`
- 已实现接口：`@Post('image')`，即 `POST /api/upload/image`
- 文件字段名：`file`
- 静态文件访问：`/uploads/*`

### 前端现状
- 报错请求：`POST http://localhost:3000/api/upload` 返回 404。
- 页面中已有一处正确示例：`<Upload action="http://<server>:3000/api/upload/image" ... />`。

### 复现步骤
1. 打开用户资料页，执行头像上传。
2. 网络面板可见请求发往 `/api/upload`。
3. 由于后端无该路由，返回 404。

### 修复建议（前端）
- 统一将所有图片/头像上传的地址改为：`POST /api/upload/image`。
- 使用 antd 的 `Upload` 组件时：
  - `action="http://<server>:3000/api/upload/image"`
  - `name="file"`（默认即为 `file`）。
- 若使用自封装 Axios：
  - URL：`/api/upload/image`
  - `FormData.append('file', file)` 提交。

### 接口约定
- Method: POST
- URL: `/api/upload/image`
- Content-Type: `multipart/form-data`
- Form 字段：`file`（仅图片，≤ 2MB）
- 响应示例：
```
{
  "url": "http://<server>:3000/uploads/1700000000000-1234567890.jpg"
}
```

### 影响范围
- 仅影响仍指向 `/api/upload` 的上传入口；改为 `/api/upload/image` 即可。
